NeuRader Manual

NeuRader is a high-performance orchestration engine written in Go. It provides a secure, push-based model for managing infrastructure without the overhead of heavy runtimes like Python or Ruby.

Installation

Run the universal installer to download the latest binary and register the systemd service.

curl -L https://neurader.operman.in/releases/neurader-latest -o neurader && chmod +x neurader && sudo ./neurader install

Node Roles

NeuRader operates in two distinct modes:

1. Jumpbox (Manager)

The control center. It holds the Master SSH keys and the inventory list. It pushes commands to child nodes via SSH.

2. Child (Agent)

The execution node. It runs the neurader daemon and waits for authorized commands from the Jumpbox.

Manual Zero-Port Setup

If your environment blocks ports 9090 and 9091, you can bypass the automated handshake entirely.

STEP 1: GET MASTER KEY

On the Jumpbox, copy the contents of the public key:

cat /etc/neurader/id_rsa.pub
STEP 2: PREPARE CHILD NODE

On the target machine, create the user and drop the key:

sudo useradd -m -s /bin/bash neurader
sudo mkdir -p /home/neurader/.ssh
echo "PASTE_KEY_HERE" | sudo tee /home/neurader/.ssh/authorized_keys
sudo chown -R neurader:neurader /home/neurader/.ssh
sudo chmod 600 /home/neurader/.ssh/authorized_keys
STEP 3: REGISTER ON JUMPBOX

Add the node to your inventory manually using the CLI:

sudo neurader add [Alias] [IP_Address]

Running Commands

Execute commands across multiple nodes in parallel:

sudo neurader run web-server,db-server "uptime"

Or list all connected hosts and their status:

sudo neurader list

File Paths

/etc/neurader/hosts.yml

The main inventory file storing node aliases and IP addresses.

/etc/neurader/id_rsa

The master private key used for orchestration. Never share this.
